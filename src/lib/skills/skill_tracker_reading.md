# SKILL CARD: Reading a Campaign Tracker
**Load this when:** A spreadsheet, Airtable export, or Google Sheet is uploaded.

---

## Step 1 — Read the Structure
Scan all tab names and column headers before doing anything else. Do not assume field names. Every brand may name columns differently.

## Step 2 — Map Columns to Concepts
Match each column to a known concept using this table. If a name is unfamiliar, infer from context.

| Concept | Common variations |
|---|---|
| Creator name | `Creator`, `Influencer`, `Handle`, `Talent`, `Name` |
| Platform | `Platform`, `Channel`, `Network`, `Social` |
| Follower count | `Followers`, `Audience Size`, `Following`, `Subs` |
| Engagement rate | `ER`, `Eng. Rate`, `Engagement %`, `Engagement Rate` |
| Views / Reach | `Views`, `Reach`, `Impressions`, `Exposure` |
| Likes | `Likes`, `Hearts`, `Reactions` |
| Comments | `Comments`, `Replies` |
| Shares / Saves | `Shares`, `Saves`, `Reposts`, `Forwards` |
| Post date | `Date`, `Post Date`, `Published`, `Go Live Date` |
| Deliverable | `Deliverable`, `Content Type`, `Format`, `Post Type` |
| Campaign status | `Status`, `Stage`, `Progress`, `State` |
| Creator rate | `Rate`, `Fee`, `Cost`, `Price`, `Compensation` |
| Offer / Negotiation | `Offer`, `Negotiated Rate`, `Counter`, `Deal` |
| Demographics | `Age`, `Gender`, `Location`, `Audience Demo` |
| Agreement | `Contract`, `Agreement`, `Signed`, `Approved` |
| Budget | `Budget`, `Total Budget`, `Allocated`, `Spend` |
| Budget split | `Split`, `Brand Split`, `Allocation %`, `Distribution` |
| Spent | `Spent`, `Actual Spend`, `Used`, `Paid` |
| Remaining | `Remaining`, `Balance`, `Left`, `Available` |

## Step 3 — Identify Tab Roles
| Tab concept | What it contains |
|---|---|
| Campaign Tracker | Dates, creator info, post performance, deliverable status |
| SOW | Offers, negotiation history, agreed rates, creator demographics |
| Contracts | Signed agreements, terms, exclusivity, usage rights |
| Dashboard | Brand budget totals, spend breakdown, budget splits |

## Step 4 — Flag Ambiguity
If a column cannot be confidently mapped, flag it and ask for clarification. Never assume or skip.
